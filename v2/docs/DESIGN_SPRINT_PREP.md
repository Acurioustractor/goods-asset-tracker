## Part 3: Consent & Cultural Protocols

### OCAP Principles (Non-negotiable)

Every piece of content involving community members or cultural knowledge requires:

1. **Ownership** - Community owns their stories
2. **Control** - Community controls how content is used
3. **Access** - Community decides who can see what
4. **Possession** - Community has copies of all content

### Consent Levels

| Level | Description | Use Cases |
|-------|-------------|-----------|
| INTERNAL ONLY | Team use only | Planning, internal comms |
| EXTERNAL-LITE | System-level, no identifiers | Aggregate stats, patterns |
| EXTERNAL | Explicit consent, culturally reviewed | Website, marketing, media |

### Elder Review Required For

- [ ] Cultural knowledge or traditional practices
- [ ] Stories from or about elders
- [ ] Content touching sacred topics
- [ ] Wisdom quotes or teachings
- [ ] Historical or genealogical information

### Consent Checklist

For each piece of content:

- [ ] Written consent obtained
- [ ] Consent scope documented (where can it be used?)
- [ ] Cultural sensitivity reviewed
- [ ] Elder review completed (if required)
- [ ] Community has copy of content
- [ ] Withdrawal process explained

## Part 6: E-commerce Requirements

### Product Data (Per Product)

```typescript
interface Product {
  // Basic
  slug: string;
  name: string;
  description: string;
  short_description: string;

  // Pricing
  price_cents: number;
  compare_at_price_cents?: number;
  currency: 'AUD';

  // Categorization
  product_type: 'basket_bed' | 'weave_bed' | 'washing_machine';

  // Media
  images: string[];  // URLs
  featured_image: string;

  // Inventory
  inventory_count: number;
  track_inventory: boolean;

  // Display
  is_active: boolean;
  is_featured: boolean;

  // Story
  artisan_id?: string;  // Link to team_members
  metadata: {
    materials?: string;
    dimensions?: string;
    weight?: string;
    assembly_time?: string;
    warranty?: string;
    environmental_impact?: string;
  };
}
```

### Order Flow

1. **Browse** → Product catalog
2. **View** → Product detail page
3. **Add** → Shopping cart
4. **Choose** → Buy for self OR Sponsor for community
5. **Checkout** → Stripe payment
6. **Confirm** → Order confirmation + email
7. **Track** → Order status updates
8. **Receive** → Delivery + impact certificate (sponsors)

### Sponsorship Flow

1. Select "Sponsor a Bed" (product or dedicated page)
2. Choose amount (bed cost + optional extra)
3. Select community (or "Where needed most")
4. Add dedication message (optional)
5. Complete payment
6. Receive impact certificate
7. Get updates when bed is delivered
8. See bed journey in dashboard
