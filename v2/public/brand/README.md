## Brand colours (from current implementation)

| Token | Hex | Use |
|-------|-----|-----|
| Cream / background | `#FDF8F3` | Page backgrounds, soft surfaces |
| Charcoal / foreground | `#2E2E2E` | Primary text, dark surfaces |
| Orange / accent | `#C45C3E` | Highlights, CTA buttons, accent rules |
| Sage / secondary | `#8B9D77` | Eyebrow text, taglines, secondary type |
| Tan / muted border | `#E8DED4` | Section dividers, card borders |

## Typography (current)

| Role | Family |
|------|--------|
| Display | Georgia, 'Times New Roman', serif |
| Body | -apple-system, BlinkMacSystemFont, 'Segoe UI', Helvetica, Arial, sans-serif |

## What proper brand work would add

A designer engagement should produce:

- True wordmark with custom typography or a refined type treatment, not Georgia browser-default
- Logo system: primary, monochrome, single-colour, knockout variants
- Full lockups (with tagline, with mark, vertical, horizontal)
- Icon set with the same visual language as the mark
- Photography style guide tied to the existing direction
- Print-ready PDF brand guide
- OG / social images using real photography from the library, not text-on-black

## Until then

Use these placeholders. They follow the brand colour and typography rules and won't actively harm the brand. They just won't help it the way a designed system would.
