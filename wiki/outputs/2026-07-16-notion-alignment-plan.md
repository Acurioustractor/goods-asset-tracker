## 4. What the hub means for pitch, proposition, and brand (plain words)

**For the latest pitch:** the hub is the spoken layer of the deck. `/pitch/deck` is what funders see; Do/Say and Start Here are what Ben says on the call ten minutes earlier. Right now the shown layer is disciplined (labels, $0-signed honesty, consent gates) and the spoken layer mostly matches, but the Artifact Hub — the page that claims to be the source of truth — would have Ben cite the wrong revenue figure and misdescribe what counts as QBE match. The single most dangerous drift is not a wrong number on a page; it is a wrong INSTRUCTION on the page Ben trusts. Once the Hub is demoted to a pointer, the pitch is coherent end to end: one number set, one label vocabulary, one honest hinge ("product proven, transfer not"), and the $0-signed admission stated identically everywhere, which is itself the credibility play with QBE.

**For the online business proposition:** "every figure audits back to goodsoncountry.com/register" is only a differentiator if it is true across surfaces. The repo makes it true by script; the hub makes it true by habit. The proposition stays: a $750 product with a falling cost curve ($685 buy-kit today → $426 in-source → ~$421 community-owned), real demand evidence (labelled requested, never committed), and a transfer pathway capital can buy. The Community OS running ahead of public canon (562 beds vs 496) is an asset — it is the pipeline proof — but only if it is fenced with a "never quote externally" banner, because a funder seeing 562 on one surface and 496 on another loses the whole audit-trail story.

**For brand and communication:** Notion is where the brand rules currently go to die — co-design on the call card, 25kg in a funder report, em dashes, a synthesis in fake quote marks, a held voice listed as usable. The fix is not more rules; it is putting the existing rules where the copying happens: the banned-word list and the six labels stamped on Do/Say and Start Here, the registry-wins pointer on every voice list, and the weekly diff pass as the enforcement. Consent stays a code gate absolutely: Notion may name cleared voices and point to the registry, but tier assignments, quotes, and the Kununurra gate live only in `storyteller-registry.ts`.

## 5. What NOT to build

- **No two-way sync.** Repo→Notion for figures, GHL→Notion for pipeline, Notion→repo read-only for the commitment register. Anything bidirectional creates a second writer and a merge problem no one will debug at 11pm before a funder call.
- **No auto-writer for money or consent.** Money stays manual sign-off (`needs-signoff.md` queue); consent data never leaves the registry via automation. The canon block script writes one page, or stages a paste — nothing more.
- **No full Notion mirror of canon.ts.** One generated block in one place. The Numbers DB keeps its genuinely additive hand-authored columns (plain-words meaning, watch-outs, who-asks) on top of synced values; it stops hand-typing the values themselves.
- **No fourth navigation layer.** Workbench is the front door; Venture Canvas links it; HQ is archive. The fix for three overlapping indexes is deletion and banners, not a new dashboard. The Sales Hub's unticked template can simply be archived.
- **No CI-integrated Notion checks.** The weekly diff pass in a Claude session is enough for a one-person team; a failing CI check on a Notion page is friction with no one to unblock it.
- **No real-time webhooks, no cron writes to Notion.** The drift window that matters is call-to-call, and the Monday loop covers it. The only automated external write stays the existing GHL rollup cron.
- **No big taxonomy migration of internal pages.** Six labels on pitch-facing surfaces only; internal working docs keep whatever shorthand is fast. Same for em dashes: sweep the Say-this blocks, ignore navigation chrome.
- **No Loop C auto-ingestion into canon.** It stays a human-review-gated report. New facts enter canon.ts by hand, with owner and asAt, or not at all.
