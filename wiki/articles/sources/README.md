# Source Register

> This is the source shelf for the Goods working pack. It shows what the source is, where a reviewer can read the accessible version, where the original should live, who should edit it, and what still needs human verification.

## Why this page matters

The wiki should not ask mentors, founders or partners to trust hidden files, code paths or old drafts. If a source matters, the useful content needs to be visible in one of three places:

- a reviewable wiki source page,
- an editable Notion review page,
- or a permissioned Drive file for the original artifact.

The wiki source page is the readable version. Notion is the human review and workshop layer. Drive is the attachment and original-document shelf.

## How to use this

- Read the source page first.
- Check the current status before using a claim in the QBE pack.
- If a source is marked sensitive, do not paste names, images, quotes or raw data into mentor material until consent and audience are checked.
- If the source is code-backed, use the wiki page for review and edit the underlying code only when the product or data truth changes.
- When a source becomes important enough for discussion, mirror it into the connected Notion workspace and record the decision back in the wiki.

## Source status language

| Status | Meaning | What to do next |
| --- | --- | --- |
| Reviewable now | A reviewer can read the useful version in this wiki. | Use it, but keep claim boundaries. |
| Move to Notion | The source needs founder comments, workshop decisions or line-by-line review. | Mirror the source note into Notion and record comments there. |
| Move to Drive | The original is a deck, PDF, spreadsheet, image folder or signed document. | Upload or link the permissioned original in Drive. |
| Sensitive | Contains story, identity, household, partner, funder or financial material. | Review consent, audience and redaction before sharing. |
| Code-backed | The source of truth is a product/data file or live system. | Translate facts into the wiki. Do not make reviewers inspect code. |
| Needs packaging | The evidence exists but is still scattered or too raw. | Create a clean source page before using it externally. |

## Rules before external sharing

- Do not share story material until consent, audience and withdrawal rights are clear.
- Do not share raw QR, asset, household or location data.
- Do not use old entity names such as ACT Foundation or ACT Ventures as current legal facts.
- Do not use bare code files as the answer. Translate the real facts into the wiki or Notion.
- Do not treat draft impact metrics as live evidence.
- Do not turn funder feedback into public copy unless the funder context is safe to share.
