## 6. What the media_links migration will link

A `media_links` table (planned, per the investor-wiki next steps) should join each media item to:
- **people**: `crm_contacts` (135 Goods contacts) and the storyteller registry, replacing the
  manifest's free-text `person` column (30 rows have one today);
- **communities**: the `communities` table rows (11 served / 12 touched), replacing directory
  naming (`images/community/<place>/`) and the manifest's `community` column (currently blank);
- **assets**: register rows (GB0-*) where a photo documents a specific delivery, e.g. the
  Maningrida machine-delivery image against the INV-0303 washers.

Best seed data, in order: `design/starred-images/_manifest.csv` (134 rows, EL refs + URLs),
`design/image-canon.json` (44 subject-to-area mappings with consent class), and
`design/deck-photos/_manifest.json` (slot-to-source paths). `local-image-tags.json` is not a
useful seed yet.
